## 方式一：Obsidian

### 什么是 Obsidian

Obsidian 是一个本地化的 Markdown 笔记应用。它的核心优势是**把你的文件夹变成一个可交互的知识图谱**——不需要上传数据到云端，所有文件都在你本地。

对于我们这个知识库来说，Obsidian 能做到：
- 用 `[[双括号链接]]` 在公司、行业、主题之间自由跳转
- 用**图谱视图**可视化所有实体之间的关系
- 用 **Dataview 插件**做结构化查询（比如"找出所有最近更新的公司"）
- 全文搜索、反向链接、标签系统开箱即用

### 安装

1. 访问 [obsidian.md](https://obsidian.md)，下载并安装（免费，Windows/Mac/Linux 都支持）
2. 安装完成后打开 Obsidian

### 打开知识库

1. 启动 Obsidian，点击 **"打开文件夹作为仓库"**（Open folder as vault）
2. 选择本项目的根目录：`C:\Users\郑曾波\Projects\company-wiki`
3. 点击确定

Obsidian 会自动读取 `.obsidian/` 目录下的配置文件，包括：
- 深色主题
- 图谱视图的颜色分组
- 自定义 CSS 样式（时间线条目、来源链接等）

> **注意**：首次打开时，Obsidian 会提示你信任仓库作者——点击"信任"即可。

### 核心功能

#### 图谱视图

图谱视图是 Obsidian 最有特色的功能，能看到所有页面之间的链接关系。

**打开方式**：左侧边栏点击 **图谱图标**（三个连线的圆点），或按快捷键 `Ctrl+G`。

你会看到：
- **绿色节点**：公司相关页面（type: company_topic）
- **蓝色节点**：行业相关页面（type: sector_topic）
- **橙色节点**：主题相关页面（type: theme_topic）
- **紫色节点**：概览页面（type: overview）

**操作技巧**：
- 滚轮缩放，鼠标拖拽平移
- 悬停节点高亮其所有连接
- 点击节点跳转到对应页面
- 左上角搜索框可过滤节点
- 右上角可切换显示/隐藏孤立节点

#### Wikilinks 跳转

知识库中大量使用 `[[双括号链接]]`，例如：

```
中微公司的新一代[[刻蚀设备]]已在[[中芯国际]]验证
```

在 Obsidian 中，点击 `[[刻蚀设备]]` 会直接跳转到刻蚀设备的行业 wiki 页面。这让你可以沿着产业链上下游自由探索。

**反向链接**：在每个页面的右侧边栏，可以看到**哪些其他页面引用了当前页面**。这对理解一个公司/行业的全局影响很有用。

#### Dataview 查询

Dataview 是 Obsidian 最强大的插件之一，可以像数据库一样查询你的 wiki。

**安装步骤**：
1. 打开 Obsidian 设置（左下角齿轮图标）
2. 进入 **社区插件** → 关闭安全模式
3. 点击 **浏览** → 搜索 **"Dataview"** → 安装 → 启用

安装后，打开项目中已有的 `examples/dataview_queries.md`，可以直接看到查询结果。

**常用查询示例**：

查找最近更新的页面：
````markdown
```dataview
TABLE last_updated AS "更新日期", sources_count AS "来源数", type AS "类型"
FROM "companies" OR "sectors" OR "themes"
WHERE last_updated
SORT last_updated DESC
LIMIT 20
```
````

按行业查看所有公司：
````markdown
```dataview
TABLE ticker AS "代码", sectors AS "行业"
FROM "companies"
WHERE file.name != "_产业链导航"
SORT file.name ASC
```
````

长期未更新的页面（超过 30 天）：
````markdown
```dataview
TABLE entity AS "实体", last_updated AS "最后更新"
FROM "companies" OR "sectors" OR "themes"
WHERE last_updated AND date(today) - date(last_updated) > dur(30 days)
SORT last_updated ASC
```
````

更多查询见 `examples/dataview_queries.md`。

#### 搜索

按 `Ctrl+Shift+F` 打开全局搜索。支持：
- 普通关键词：`中微公司`
- 引号精确匹配：`"刻蚀设备"`
- 标签搜索：`tag:#半导体设备`
- 组合搜索：`中微公司 刻蚀`

### 推荐插件

在设置 → 社区插件中搜索安装：

| 插件 | 用途 |
|------|------|
| **Dataview** | 结构化查询（必装） |
| **Calendar** | 日历视图，按日期浏览更新 |
| **Recent Files** | 快速访问最近打开的文件 |
| **Tag Wrangler** | 管理和重命名标签 |
| **Quiet Outline** | 页面目录导航（大纲视图） |

### Web Clipper 剪藏

Obsidian 提供浏览器插件 [Web Clipper](https://obsidian.md/clipper)，可以把网页文章一键保存到知识库。

**配置方法**：
1. 在浏览器中安装 Obsidian Web Clipper 扩展
2. 打开扩展设置，配置保存路径为：`companies/_inbox/{{title}}.md`
3. 模板内容设置为：

```markdown
---
title: "{{title}}"
source: "{{url}}"
published_date: "{{date|date:'YYYY-MM-DD'}}"
type: news
---
# {{title}}

{{content}}
```

以后浏览新闻时，点击 Web Clipper 图标即可将文章保存到 `companies/_inbox/` 目录，等待 LLM 整理。

### 常见问题

**Q: 为什么有些 `[[链接]]` 显示为灰色？**
A: 灰色表示目标页面不存在（"未解析链接"）。这通常是因为某个实体被引用但没有创建 wiki 页面。

**Q: PDF 文件能直接在 Obsidian 中打开吗？**
A: 可以。设置中已启用 `showUnsupportedFiles`，在文件树中可以看到 PDF 文件并双击打开。

**Q: 图谱太密集看不清怎么办？**
A: 在图谱视图左上角的筛选框中输入关键词过滤。也可以在图谱设置中调整节点大小和力的参数。

## 方式二：Web 浏览器

Web 方式用 MkDocs Material 生成一个静态网站，适合：
- 不想安装 Obsidian 时快速查看
- 在手机/平板上浏览
- 分享给他人

### 快速开始

**方法一：直接打开已构建的站点**

如果之前已经运行过构建，直接双击打开：

```
C:\Users\郑曾波\Projects\company-wiki\web\dist\index.html
```

**方法二：重新构建**

如果知识库内容有更新，需要重新构建：

```bash
cd C:\Users\郑曾波\Projects\company-wiki
python web/preprocess.py
```

构建过程会：
1. 扫描所有 wiki 页面（公司/行业/主题）
2. 将 `[[wikilinks]]` 转换为标准链接
3. 从 `graph.yaml` 导出图谱数据
4. 生成静态网站到 `web/dist/`

首次运行时会自动下载 vis.js 图谱组件（约 200KB），需要网络连接。后续构建离线可用。

### 浏览页面

打开后你会看到：
- **顶部导航栏**：首页 / 产业链 / 公司 / 行业 / 主题 / 图谱
- **左侧边栏**：当前分类下的所有页面
- **右侧**：页面正文

每个公司的页面包含核心问题、时间线（按日期倒序）、综合评估。点击页面中的链接可以在不同公司/行业之间跳转。

### 搜索

点击顶部导航栏的 **搜索图标**（放大镜），或按 `Ctrl+K`（在 Mac 上 `Cmd+K`）。

支持中文搜索。搜索结果会高亮关键词在页面中的位置。

### 产业链图谱

点击顶部导航的 **"图谱"** 标签，打开交互式产业链图谱。

**图谱功能**：
- **拖拽节点**：鼠标拖拽调整节点位置
- **滚轮缩放**：放大/缩小视图
- **点击节点**：查看实体详情，公司节点可直接跳转到对应 wiki
- **搜索过滤**：左上角搜索框输入关键词，匹配的节点高亮，其余变淡
- **类型过滤**：下拉菜单选择只显示公司/行业/子行业/主题
- **重置视图**：点击"重置视图"按钮回到初始状态

**节点颜色含义**：
| 颜色 | 含义 |
|------|------|
| 橙色大圈 | 主题（如 AI产业链） |
| 蓝色中圈 | 行业（如 半导体设备） |
| 浅蓝色小圈 | 子行业（如 刻蚀设备） |
| 绿色小圈 | 公司（如 中微公司） |

**连线含义**：
| 样式 | 含义 |
|------|------|
| 蓝色实线箭头 | 上游供应关系（from 是 to 的上游） |
| 浅蓝虚线箭头 | 归属关系（子行业属于行业） |
| 绿色细线箭头 | 公司所属行业 |
| 红色虚线 | 竞争关系 |

### 重新构建

当知识库内容更新后，运行以下命令重新构建 Web 站点：

```bash
cd C:\Users\郑曾波\Projects\company-wiki
python web/preprocess.py
```

如果只想更新导航但不重新构建（比如只新增了一两个页面），可以修改 `preprocess.py` 中的最后一步。不过一般直接运行完整构建即可——127 个页面的构建非常快。

### 开发模式

如果你需要调试或实时预览修改效果：

```bash
cd C:\Users\郑曾波\Projects\company-wiki\web
mkdocs serve
```

然后在浏览器打开 http://localhost:8000。修改 `docs/` 下的文件后页面会自动刷新。

> 注意：`mkdocs serve` 读取的是 `docs/` 目录下的文件。如果你修改了原始 wiki 内容，需要先运行 `python web/preprocess.py` 来更新 `docs/` 目录。

## 两种方式对比

| 维度 | Obsidian | Web 浏览器 |
|------|----------|-----------|
| **安装** | 需安装 Obsidian 应用 | 无需安装，双击 HTML 即可 |
| **实时编辑** | 可以直接编辑 wiki 文件 | 只读浏览 |
| **图谱视图** | 全局关系图（基于 wikilinks） | 产业链结构图（基于 graph.yaml） |
| **搜索** | 全文 + 正则 + 标签 | 全文（内置中文分词） |
| **Dataview** | 支持 SQL 式查询 | 不支持 |
| **反向链接** | 每个页面右侧显示 | 不支持 |
| **移动端** | Obsidian 移动端 App | 浏览器直接访问 |
| **分享** | 不方便 | 直接发送 dist/ 文件夹 |
| **离线** | 完全离线 | 构建后完全离线 |

**建议**：日常深入研究用 Obsidian，快速查阅或手机浏览用 Web 站点。每次 LLM 更新 wiki 后，运行一次 `python web/preprocess.py` 同步 Web 站点。
